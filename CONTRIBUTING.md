## Attribution and Credit

CODAI is an open source project created by **Arian Rudd**. When contributing, please:

1. Respect the original authorship and branding
2. Maintain references to "CODAI" and "Arian Rudd" in appropriate places
3. Do not remove existing copyright notices or attribution

## Code of Conduct

By participating in this project, you agree to:

- Be respectful and inclusive
- Welcome newcomers and help them get started
- Focus on what is best for the community
- Show empathy towards other community members

## How Can I Contribute?

### Reporting Bugs

Before creating bug reports, please check existing issues. When creating a bug report, include:

- A clear and descriptive title
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- System information (OS, Python version, Node.js version)
- Any relevant logs or screenshots

### Suggesting Enhancements

Enhancement suggestions are welcome! Please provide:

- A clear and descriptive title
- A detailed description of the proposed enhancement
- Any possible implementation approaches
- Why this enhancement would be useful to most users

### Pull Requests

1. Fork the repo and create your branch from `main`
2. If you've added code that should be tested, add tests
3. Ensure the test suite passes
4. Make sure your code follows the existing style
5. Include appropriate documentation updates
6. Add yourself to CONTRIBUTORS.md (create if doesn't exist)

## Development Setup

1. Follow the setup instructions in README.md
2. Install development dependencies
3. Run tests before submitting PR

## Style Guidelines

### Python Code
- Follow PEP 8
- Use type hints where appropriate
- Add docstrings to functions and classes
- Keep functions focused and small

### JavaScript/TypeScript Code
- Use TypeScript for new code
- Follow the existing ESLint configuration
- Use functional components for React
- Include proper TypeScript types

### Commit Messages
- Use clear and meaningful commit messages
- Start with a verb (Add, Fix, Update, etc.)
- Keep the first line under 50 characters
- Reference issues and pull requests when relevant

## Attribution in Your Work

If you create derivative works based on CODAI:

1. Include "Based on CODAI by Arian Rudd" in your documentation
2. Link back to the original repository
3. Maintain the GPL-3.0 open source license

## Recognition

Contributors who make significant contributions will be:
- Added to the CONTRIBUTORS.md file
- Mentioned in release notes
- Thanked in the project documentation

## Questions?

Feel free to open an issue for any questions about contributing.
